# 한국어 응답 규칙

## 언어 설정

- 모든 응답은 한국어로 작성합니다.
- 코드 주석도 가능한 한 한국어로 작성합니다.
- 기술 용어는 필요시 영어와 한국어를 병행 표기합니다 (예: "컨테이너(container)").
- 에러 메시지나 로그는 원본 언어를 유지하되, 설명은 한국어로 제공합니다.

## 예외 상황

- 코드 자체는 영어로 작성합니다 (변수명, 함수명, 클래스명 등).
- 공식 문서 인용, 명령어, 라이브러리 API 이름은 원본 언어를 유지합니다.
- 사용자가 명시적으로 다른 언어를 요청하는 경우에만 예외를 적용합니다.

## 코드 주석 예시

```javascript
// 사용자 데이터를 가져오는 함수
function getUserData(userId) {
  // API 호출을 통해 사용자 정보 조회
  return fetch(`/api/users/${userId}`)
    .then(response => response.json())
    .catch(error => {
      // 에러 처리: 사용자 데이터 로드 실패
      console.error('사용자 데이터 로드 중 오류 발생:', error);
    });
}
```

## 문서화 스타일

- README, 디자인 문서, 가이드 문서는 한국어로 작성합니다.
- 기술적 설명은 명확하고 이해하기 쉽게 작성합니다.
- 코드 예제에는 한국어 설명을 포함합니다.
- 단, 오픈소스 배포가 목적인 README/CHANGELOG 는 영어를 우선하고 한국어를 보조로 둡니다.

## 커밋 메시지 / PR 설명

- 팀 컨벤션이 없으면 커밋 메시지 본문은 한국어로 작성합니다.
- Conventional Commits 의 `type:` 접두어 (feat, fix, refactor 등) 는 영어를 유지합니다.
- PR 설명의 "Summary" / "Test plan" 같은 머리말은 영어 그대로 두되 항목은 한국어로 작성합니다.
